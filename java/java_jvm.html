# JVM

    # JVM（Java Virtual Machine）

    ​ JVM是一种用于计算设备的规范，它是一个虚构出来的计算机，是通过在实际的计算机上仿真模拟各种计算机功能来实现的。

    ​ JVM的优势在于Java的程序在一次编译以后可以在不同的平台上运行，Java语言通过虚拟机调用实体机的各种指令，而不同平台的JVM适配了Java字节码和当前平台的指令集，就是常说的“一次修改，到处运行”。

    ​ Java虚拟机有自己完善的硬件架构，如处理器、堆栈等，还具有相应的指令系统。Java虚拟机主要分为五大模块：类装载器子系统、运行时数据区、执行引擎、本地方法接口和垃圾收集模块。其中垃圾收集模块在Java虚拟机规范中并没有要求Java虚拟机垃圾收集，但是在没有发明无限的内存之前，大多数JVM实现都是有垃圾收集的。

    总结：JVM是使Java程序在不同平台上只要有运行环境都可以运行的虚拟机

    # JRE： Java Runtime Environment

    ​ java运行时环境。JRE是一个软件，是由Sun公司研发的可以让计算机运行Java程序的软件，JRE内部有一个Java虚拟机，也就是我们常常听到的JVM(Java Virtual Machine)，以及一节标准的类别函数库(Class Library)

    总结：JRE是使用java语言编写的程序运行所需要的软件环境 JDK：Java Development Kit ​ Java 标准开发工具包，包含JRE。用户只需要安装JRE来运行Java程序，而开发者必须安装JDK来开发程序。他提供了编译器，运行环境和常用的基础库。

    # JDK有三种版本

    标准版 J2SE(java2 Standard Edition) 包含那些构成Java语言核心的类。比如：数据库连接、接口定义、输入/输出、网络编程 企业版 J2EE(Java 2 Enterprise Edition) Java产生于1995年，当时的主要用途是制作动态网页的Applet，后来人民发现Java的“一次开发，到处运行”和纯OO（面向对象）的特性，以及垃圾回收机制和内置安全特别适合开发企业应用系统，于是Sun联合IBM、Oracle、BEA等大型企业应用系统开发商与1999年沟通制定了J2EE开发规范。 J2EE定义了动态Web页面功能（Servlet和Jsp）、商业组件（EJB）、异步消息传输机制（JMS）、名称和目录定位服务（JNDI）、事务控制、数据库访问（JDBC）、与子系统的连接器（JCA）和安全服务等。 微缩版 J2ME(Java 2 Micro Edition) 为机顶盒、移动电话和PDA之类嵌入式消费电子设备提供的Java语言平台，包括虚拟机和一系列标准化的Java API。 总结：JDK是java开发工具包，是程序员使用java语言编写java程序所需的开发工具包
